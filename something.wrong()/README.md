## 需求分析的`非非`

- 需求理解偏差
- 场景遗漏





## 需求设计的`非非`

- 方案欠佳
- 设计形式主义
- 抽象不到位
- 数据结构欠佳
- 耦合中
- 无DFX设计
- 忽略程序性能
- 过度设计
- 过分乐观
- 无文档
- 文档不共享





## 开发的`非非`

- 代码编写的非非
  - 命名不当
  - 低级拼写错误
  - 复制粘贴错误
  - 低效的实现
  - 注释不当
- 一刀切的军规
- 代码提交的非非
  - commit信息随意
  - 代码不上库
  - 提交过多代码
- 重构的非非
  - 代码腐化不重构
  - 不想/敢重构

- 抽象和封装不够





## UT&ST的`非非`

- 无UT/ST
- 不重视UT/ST
- 场景遗漏
- 后补UT
- 片面相信覆盖率
- UT/ST代码随意
- 侵入源码做适配
- 实现成本过高
- 换人写UT
- 未发现问题
